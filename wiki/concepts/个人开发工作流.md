# 个人开发工作流

## 用户背景

- 编程了解但不深刻
- 逻辑能力强，知道基本开发逻辑
- 有 idea 和想法，需要完整方法论保证流程可控

## 核心需求

1. **记忆系统** - 记录所有核心记忆内容
2. **总控系统** - 协助实现 idea
3. **Research 系统** - 调研、整合信息
4. **图形化界面** - 可视化正在做什么、做什么、准备做什么
5. **开发工具链** - Claude Code, Codex, Gemini CLI
6. **部署能力** - 阿里云、腾讯云

## 各系统职责

| 系统 | 职责 | 状态 |
|------|------|------|
| OpenViking | 记忆存储、上下文管理 | 已部署在 192.168.5.225 |
| OpenHarness | Agent 编排、工具调用 | 待部署 |
| gstack Skills | 规划审查、QA | 已安装 |
| Deer Flow | 研究流、sub-agent | 本地已有 |
| Claude/Codex/Gemini | 编码执行 | CLI 已安装 |

## 开发流程

```
1. Idea → gstack /plan-ceo-review (产品视角)
                    ↓
2. 规划 → gstack /plan-eng-review (架构视角)
                    ↓
3. Research → Deer Flow (调研整合)
                    ↓
4. 知识积累 → OpenViking (记忆存储)
                    ↓
5. 执行 → Claude/Codex (编码)
                    ↓
6. QA → gstack /qa (测试)
                    ↓
7. 部署 → 阿里云/腾讯云
```
